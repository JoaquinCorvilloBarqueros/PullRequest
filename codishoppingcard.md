Una Pull Request (PR) es la petición formal para fusionar una rama con la rama principal (main).
**Para qué sirve:** permitir revisión de código, pruebas y discusión antes de integrar cambios.
**Ventajas:**
- Revisión colaborativa y control de calidad.
- Registro histórico y discusión.
- Prevención de errores en la rama main.
**Desventajas:**
- Requiere proceso y coordinación (puede ralentizar el flujo si hay demasiada burocracia).
- Posible necesidad de resolver conflictos.
